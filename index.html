This application is to show the comparative speed and logic of three popular sorts with arrays. Bubble sort is a solution many naive programmers begin with. Compare every element to its neighbor and switch if needed. However that takes a lot of time and with large data sets becomes unmanageable. Merge sort takes the approach of divide and conquer by sorting sections of the array one at a time. However this causes the need for an auxiliary "sorted" array. That takes up too much space if you have elements that contain lots and lots of data. In place quick sort can be faster than merge sort - or as slow as bubble sort - depending on its initial state and pivot choice but on average it is the same speed as merge sort with out the need for extra space.
